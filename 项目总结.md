## 功能概述

基于 LangGraph 构建的多 Agent 系统，模拟电子产品电商场景：

| Agent | 角色 | 职责 |
|-------|------|------|
| Supervisor（调度层） | 总控 | 意图识别 + 任务路由 |
| Sales Agent（销售顾问） | 售前 | 产品推荐、价格查询 |
| Service Agent（售后客服） | 售后 | 订单查询、退货/换货处理 |
| Tech Agent（技术支持） | 技术 | 故障诊断、使用问题 |
| Chat Agent（闲聊助手） | 引导 | 问候、引导、模糊意图兜底 |

## 技术亮点

1. **LangGraph 有状态图**：节点函数返回 dict 自动更新状态（Pydantic v2 兼容）
2. **条件路由**：`add_conditional_edges` 根据意图自动分流
3. **可扩展架构**：新增 Agent 只需 `add_node` + 一行路由映射
4. **Mock 兜底**：无 API Key 也可运行，`USE_REAL_LLM` 开关一键切换
5. **规则引擎优先**：意图分类先用关键词规则精确匹配，无法判断时 fallback 到 LLM

## 与项目一（RAG 知识库）的互补性

| 对比维度 | 项目一：RAG 知识库 | 项目二：Multi-Agent |
|---------|------------------|-------------------|
| 核心能力 | 理解（检索 + 生成） | 行动（编排 + 工具） |
| 技术栈 | Streamlit + BM25 | LangGraph + FastAPI |
| 架构 | 单一 RAG 链路 | 多 Agent 状态图 |
| 数据来源 | JSON 文件持久化 | 内存 Mock 数据 |
| 面试方向 | 检索、Embedding | Agent 编排、工具调用 |
