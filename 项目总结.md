## 功能概述

基于 LangGraph 构建的多 Agent 系统，模拟电子产品电商场景：

1. **Supervisor（调度层）**：意图识别 + 任务路由
2. **Sales Agent（销售顾问）**：产品推荐、价格查询
3. **Service Agent（售后客服）**：订单查询、退货/换货处理
4. **Tech Agent（技术支持）**：故障诊断、使用问题
5. **Chat Agent（闲聊助手）**：问候、引导

## 技术亮点

1. **LangGraph 有状态图**：节点函数返回 dict 自动更新状态（Pydantic v2 兼容）
2. **条件路由**：`add_conditional_edges` 根据意图自动分流
3. **可扩展架构**：新增 Agent 只需 `add_node` + `add_edge`
4. **Mock 兜底**：无 API Key 也可运行，`USE_REAL_LLM` 开关一键切换
5. **相对导入→绝对导入**：解决包层级问题

## 面试话术

**"为什么用 LangGraph 而非常见的多Agent框架？"**
LangGraph 提供有向状态图，可以精确控制 Agent 之间的交互流程和状态共享，比 LangChain Agent 更灵活。每个节点拿到完整状态上下文，返回增量更新，天然支持并行和条件分支。

**"如何保证可扩展性？"**
架构设计是"Supervisor + 专业 Agent"模式。新增 Agent 只需写一个节点函数、在 workflow 里 `add_node`、在路由映射里加一行。零侵入，不需要改任何现有逻辑。

**"Mock 模式的意义？"**
面试时可能没有 API Key。系统在 `USE_REAL_LLM=False` 时用规则引擎做意图分类和回复生成，功能完整可演示。设置环境变量后就自动切换真实 LLM，无缝过渡。
